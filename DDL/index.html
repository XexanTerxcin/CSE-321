Data Link Layer in OSI Model

The Data Link Layer is the heartbeat of local network communication — raw bits transform into structured frames, errors vanish, and medium access is orchestrated with precision.

  • 📦 Framing: Organizes bits into addressable frames.
  • 🛡️ Error Control: CRC & checksums ensure integrity.
  • 🌊 Flow Control: Prevents data flood between devices.
  • 🎛️ MAC: Coordinates who speaks on the shared channel.

⚙️ Core Jobs of Data Link Layer

  1. Framing → Encapsulation of packets into frames with headers & trailers.
  2. Error detection/correction → FCS, CRC, ARQ mechanisms.
  3. Flow control → Sliding window, backpressure.
  4. Media Access Control → CSMA/CD, CSMA/CA, Token passing.

🎯 Error Control Protocols

⏸ Stop-and-Wait ARQ

  • Simplest: send frame → wait ACK → timeout resend. Reliable but low throughput.

⚡ Sliding Window Protocol

  • Sender window of N frames; pipeline transmission, Go-Back-N & Selective Repeat increase efficiency exponentially.
  • Analogy: High-speed rail of data packets with smart acknowledgments.

📡 Media Access Control & MAC Addressing

MAC (Media Access Control) sublayer manages channel arbitration & collision avoidance. Each network interface gets a 48-bit unique MAC address (e.g., 00:1A:2B:3C:4D:5E).

  • Wired: CSMA/CD (Collision Detection)
  • Wireless: CSMA/CA (Collision Avoidance) plus RTS/CTS handshake.
  • Wi-Fi frames carry source/destination MACs, ensuring delivery in wireless cells.

Without MAC, networks would collapse into chaos — the true traffic conductor of Layer 2.
