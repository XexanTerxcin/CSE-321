Data Link Layer in OSI Model

The Data Link Layer is the second layer of the OSI model.

  • It takes raw data from the physical layer (signals, bits) and organizes it into frames (packets with structure).
  • It makes sure the data is delivered error-free from one device to another on the same network.
  • It also controls who gets to use the network medium (this is where MAC – Media Access Control comes in).

Main jobs of the Data Link Layer:

  1. Framing → Wraps raw bits into structured frames.
  2. Error detection/correction → Checks if data got damaged and fixes or resends it.
  3. Flow control → Ensures devices don’t overwhelm each other with too much data.
  4. MAC (Media Access Control) → Decides which device can send data at a given time.

Protocol in Data Link Layer:

In the Data Link Layer, a protocol is simply the set of rules that control how data is packaged, transmitted, and checked for errors between devices on the same network link.

Framing & Access Protocols:

  • Ethernet (IEEE 802.3) → Standard for wired LANs.
  • Wi-Fi (IEEE 802.11) → Standard for wireless LANs.
  • PPP (Point-to-Point Protocol) → Used for direct connections.
  • HDLC (High-Level Data Link Control) → Reliable communication in WANs.
  • SDLC (Synchronous Data Link Control) → IBM’s protocol for synchronous communication, predecessor of HDLC.
  • SLIP (Serial Line Internet Protocol) → Simple protocol for transmitting IP packets over serial lines.
  • LAP (Link Access Procedure) → Used in X.25 networks for reliable data transfer.
  • LCP (Link Control Protocol) → Part of PPP, establishes and configures data link connections.
  • NCP (Network Control Protocol) → Works with PPP to configure different network layer protocols (like IP).

Error Control Protocols:

Stop-and-Wait ARQ

  • Send one frame, wait for acknowledgment (ACK).
  • If no ACK → resend.
  • Simple but slow.
  • Analogy: Sending one text message and waiting for “Got it!” before sending the next.

Sliding Window Protocol

  • General method behind Go-Back-N and Selective Repeat.
  • Sender has a “window” of frames it can send before needing ACKs.
  • Keeps the channel busy and avoids waiting too much.
  • Improves efficiency by balancing speed and reliability.
  • Analogy: Like delivering multiple parcels at once, but keeping track of which ones got signed for.
📡 WI-FI (IEEE 802.11) — WIRELESS MEDIA ACCESS CSMA/CA

Media Access Control :

MAC = Media Access Control It’s part of the Data Link Layer in networking. MAC is the rule-set that controls how devices share the network fairly.

  • It decides which device gets to send data at a given time.
  • It prevents “collisions” (two devices talking at once).
  • It uses MAC addresses to identify devices.

MAC Address:

  • A MAC Address (Media Access Control Address) is a unique hardware ID given to a device’s network card.
  • Written in hexadecimal (like 00:1A:2B:3C:4D:5E).

Wireless networks rely on CSMA/CA for efficient medium sharing. Wi-Fi (IEEE 802.11) defines how wireless devices communicate at the Data Link Layer, using frames and MAC addresses just like Ethernet, but with additional mechanisms to avoid collisions over radio.
